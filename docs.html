Overview

Vessel follows the classic build, release, run model with a transparent audit trail.

  • Build: detect stack, compile, and cache dependencies.
  • Release: run release checks and version immutable slugs.
  • Run: launch web and worker processes with health checks.
  • Observe: stream logs, metrics, and release events.

Quickstart

Deploy a service from a local repository in minutes.

$ vessel login
$ vessel init
$ git push vessel main
$ vessel releases

CLI Commands

  • vessel login - authenticate to the control plane.
  • vessel init - create an app and attach a pipeline.
  • vessel releases - list release history.
  • vessel rollback v12 - roll back to a previous release.
  • vessel logs -t - stream logs in real time.

Deploy Flow

Each deploy follows the same observable pipeline.

  • Push code or connect a Git provider.
  • Buildpack compiles the artifact and stores the slug.
  • Release phase runs migrations or checks.
  • Traffic shifts to the new release.

Config Manifest

Define processes and health checks in a compact manifest.

app: vessel-starter
stack: nodejs
processes:
  web: npm start
  worker: npm run jobs
healthchecks:
  /health: 200

Runtime And Scaling

Scale web or worker processes independently and keep resources predictable.

  • Scale replicas with explicit resource quotas.
  • Define process types in the manifest or Procfile.
  • Use release phase hooks to gate migrations.

Logs And Monitoring

Stream logs and trace releases with a single command.

$ vessel logs -t
$ vessel events
$ vessel metrics

FAQ

Can I deploy multiple pipelines?

Yes. Each app can have multiple pipelines for dev, staging, and production.

Do I need a Procfile?

Only if you want to override the detected start command or add worker processes.

How are rollbacks handled?

Rollbacks point to a prior immutable release without rebuilding.
